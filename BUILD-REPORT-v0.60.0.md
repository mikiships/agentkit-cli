### Regression notes

The 18 failures are all **pre-existing D5 tests from earlier build contracts** that hardcode specific version strings (e.g., `assert 'version = "0.59.0"' in content`). These tests fail whenever the version is bumped because they check their own contract's target version. This is a pre-existing test suite design issue — each version's D5 test checks for that exact version number, so bumping to 0.60.0 breaks them.

Files affected:
- `test_benchmark_d5.py`, `test_certify_d5.py`, `test_daily_d5.py`, `test_daily_pages_d5.py`
- `test_digest_d5.py`, `test_explain.py`, `test_improve.py`, `test_landing_d5.py`
- `test_migrate_d5.py`, `test_monitor_d5.py`, `test_org_pages_d5.py`, `test_search_d5.py`
- `test_timeline_d5.py`, `test_trending_pages_d5.py` (+ 4 others)

These were passing before the version bump and will fail again when any future version bump occurs. They are not regressions introduced by this feature.

## Issues / Notes

1. **Version hardcoded tests**: Pre-existing test suite design issue (see above). Not introduced by this build.
2. **`analyze_target` integration**: `_analyze_repo()` calls `analyze_target()` directly. In production this requires tools (agentmd, agentlint, etc.) to be installed. Tests bypass this with `_analyze_override` dependency injection.
3. **GitHub token**: Context file checks via Contents API will consume rate limit on unauthenticated requests. `GITHUB_TOKEN` recommended for >20 repos.
